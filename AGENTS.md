# AGENTS.md

## О проекте

- Репозиторий содержит книгу `"От чёрного ящика к инженерии"` о механике LLM, промпт-протоколах, агентных контурах и AI-friendly разработке.
- Основной редактируемый корпус книги находится в `book/`: главы разбиты на отдельные Markdown-файлы с числовыми префиксами (`00_...md`, `01_...md`, ... `16_...md`).
- Сборка PDF идёт из `book/` через `scripts/build_book_pdf.py`. Обёртка для локального запуска находится в `build_pdf.command`.
- Артефакт `BlackboxBook.pdf` не редактируется вручную.
- Файл `От чёрного ящика к инженерии.md` выглядит как сводная версия рукописи. Для изменений по содержанию ориентируйся прежде всего на `book/`, если задача явно не требует синхронизации сводного файла.

## Как писать и редактировать

- Язык книги: русский. Английский допустим для терминов, названий методов, API, статей и примеров кода.
- Тон: инженерный, точный, насыщенный смыслом, без маркетинговой риторики и без разговорной воды.
- Объяснения должны быть практичными: не только "что это", но и "почему это важно" и "что с этим делать".
- Предпочтительный стиль уже виден в главах: аналогия -> механизм -> практические следствия -> источники.
- Не упрощай текст до блогового уровня и не превращай его в академическую статью. Это книга для практикующих инженеров.

## Структура глав

- Сохраняй существующий формат глав: заголовок верхнего уровня, секции `##`, подсекции `###`, разделители `---` там, где они уже используются.
- Почти каждая глава заканчивается блоками `Практический вывод`, `Источники` и `Навигация`. При содержательных правках сохраняй этот паттерн.
- Навигационные ссылки между главами должны оставаться корректными, особенно если меняются заголовки, порядок или имена файлов.
- Имена файлов в `book/` отражают порядок чтения. Новые главы или разделы добавляй только с сохранением последовательной нумерации.

## Содержательные правила

- Сохраняй терминологическую консистентность: LLM, attention, MLP, MoE, RAG, RLHF, DPO, structured outputs, tool use, agent loop.
- Если добавляешь новые утверждения о моделях, архитектурах или индустрии, опирайся на проверяемые первоисточники, а не на общие формулировки.
- Для прикладных рекомендаций предпочитай конкретные паттерны, анти-паттерны, чек-листы и короткие примеры кода.
- Формулы, таблицы и списки используй там, где они реально повышают ясность.
- Не удаляй намеренно выразительные сравнения и метафоры, если они помогают объяснению и уже соответствуют тону книги.

## Источники и фактичность

- Раздел `Источники` должен содержать первичные статьи, репозитории, бенчмарки или официальную документацию.
- Для фундаментальных тем допустимы более ранние работы, но контекст книги ориентирован на 2023-2026 годы.
- Не добавляй сомнительные факты, проценты и рыночные оценки без возможности их защитить источником.

## Сборка и проверка

- Основная команда сборки: `python3 scripts/build_book_pdf.py --source book --output BlackboxBook.pdf`.
- Альтернативный локальный запуск: `./build_pdf.command`.
- Для сборки нужны `pandoc` и один из PDF-движков: `tectonic`, `xelatex` или `lualatex`.
- Скрипт сборки санитизирует часть Unicode-символов и табличных обозначений только во временной копии. Исходные Markdown-файлы должны оставаться читаемыми для людей.

## Чего не делать

- Не редактируй `BlackboxBook.pdf` вручную.
- Не меняй структуру книги, нумерацию глав и навигацию без явной причины.
- Не вноси стилистические массовые правки ради "унификации", если они не улучшают точность, читаемость или согласованность.
- Не засоряй текст лишними комментариями для редакторов внутри глав, если это не было явно запрошено.
